# 简单编程保存流程说明

## 重要发现

**简单编程界面的保存功能存在断电保持不完全的问题！**

虽然简单编程界面调用了与通用编程相同的 `saveProgram()` 函数，但由于设计上的缺陷，只有程序指令被持久化，而简单编程界面特有的槽位数据（`stepSlots`）并未被保存。这导致：

1. ✓ 程序指令可以断电保持（可在通用编程界面查看）
2. ✗ 简单编程界面的编辑状态无法恢复
3. ✗ 断电后无法继续在简单编程界面编辑

详见本文档第九章"总结"部分的详细分析。

---

## 概述

本文档详细说明简单编程界面（Easy Program Wizard）中"保存"按钮的执行流程，并与通用编程界面的保存流程进行对比分析，揭示断电保持不完全的根本原因。

## 二、保存流程详解

### 2.1 入口函数：`quickPage.save()`

**文件**: `qml/App_HAMOUI/teach/EasyProgramQuickPage.qml`

```javascript
function save() {
    var missings = missingSlots();
    var programInstance;
    var targetProgramIndex;
    var programModel;
    var orderedSlots;
    var program;
    var i;

    // 步骤1：验证模板定义
    if(!currentTemplateDefinition) {
        tipDialog.warning(qsTr("该模板快设暂未实现"), qsTr("OK"));
        return;
    }

    // 步骤2：检查槽位完整性
    if(missings.length > 0) {
        tipDialog.warning(qsTr("槽位未全部插入指令"), qsTr("OK"));
        return;
    }

    // 步骤3：获取程序实例
    programInstance = Teach.programInstance;
    if(!programInstance) {
        tipDialog.warning(qsTr("程序页面尚未就绪"), qsTr("OK"));
        return;
    }

    // 步骤4：获取目标程序索引和模型
    targetProgramIndex = programInstance.currentProgramIndex();
    programModel = ProgramFlowData.programs[targetProgramIndex];
    if(!programModel) {
        tipDialog.warning(qsTr("目标程序模型不存在"), qsTr("OK"));
        return;
    }

    // 步骤5：按步骤索引排序槽位
    orderedSlots = stepSlots.slice(0);
    orderedSlots.sort(function(a, b) {
        return a.stepIndex - b.stepIndex;
    });
    
    // 步骤6：构建程序数组
    program = [];
    for(i = 0; i < orderedSlots.length; ++i) {
        program.push(cloneObject(orderedSlots[i].actionObject));
    }
    program.push(Teach.generateEndAction());  // 添加结束指令

    // 步骤7：更新程序模型
    programInstance.updateProgramModel(programModel, program);
    
    // 步骤8：收集特殊行信息
    programInstance.collectSpecialLines(targetProgramIndex);
    
    // 步骤9：整理编号
    programInstance.onFixIndexTriggered();
    
    // 步骤10：标记为已修改
    programInstance.hasModify = true;
    
    // 步骤11：保存程序
    programInstance.saveProgram(targetProgramIndex);
}
```

## 三、关键步骤详解

### 3.1 步骤1-2：前置验证

#### 验证模板定义
```javascript
if(!currentTemplateDefinition) {
    tipDialog.warning(qsTr("该模板快设暂未实现"), qsTr("OK"));
    return;
}
```

**说明**: 确保当前模板已正确加载和定义。

#### 检查槽位完整性
```javascript
if(missings.length > 0) {
    tipDialog.warning(qsTr("槽位未全部插入指令"), qsTr("OK"));
    return;
}
```

**说明**: 
- `missingSlots()` 函数检查所有必填槽位是否都已插入指令
- 如果有未填充的槽位，阻止保存并提示用户

### 3.2 步骤3-4：获取程序实例和模型

#### 获取程序实例
```javascript
programInstance = Teach.programInstance;
```

**说明**: 
- `Teach.programInstance` 指向 `ProgramFlowPage` 实例
- 在 `ProgramFlowPage.qml` 的 `Component.onCompleted` 中设置：
  ```javascript
  Teach.programInstance = programFlowPageInstance;
  ```

#### 获取目标程序索引
```javascript
targetProgramIndex = programInstance.currentProgramIndex();
```

**说明**: 
- 获取当前正在编辑的程序索引
- 0 = 主程序，1-16 = 子程序

#### 获取程序模型
```javascript
programModel = ProgramFlowData.programs[targetProgramIndex];
```

**说明**: 
- `ProgramFlowData.programs` 是存储所有程序的 ListModel 数组
- 每个程序对应一个 ListModel

### 3.3 步骤5-6：构建程序数组

#### 排序槽位
```javascript
orderedSlots = stepSlots.slice(0);
orderedSlots.sort(function(a, b) {
    return a.stepIndex - b.stepIndex;
});
```

**说明**: 
- `stepSlots` 是模板快设页面中的槽位数组
- 按 `stepIndex` 排序，确保指令按正确顺序执行

#### 构建程序
```javascript
program = [];
for(i = 0; i < orderedSlots.length; ++i) {
    program.push(cloneObject(orderedSlots[i].actionObject));
}
program.push(Teach.generateEndAction());
```

**说明**: 
- 从每个槽位中提取 `actionObject`（动作对象）
- 克隆对象，避免引用问题
- 最后添加结束指令（`Teach.generateEndAction()`）

### 3.4 步骤7：更新程序模型

**文件**: `qml/App_HAMOUI/teach/ProgramFlowPage.qml`

```javascript
function updateProgramModel(model, program){
    model.clear();  // 清空现有模型
    var step;
    var at;
    var isSyncStart = false;
    var isPSStart = false;
    var jumpLines = [];
    var insertedIndex = 0;
    var currentParsingProgram = PData.programToParsingIndex(model);
    
    // 清除标志定义
    Teach.currentRecord.flagsDefine.clear(currentParsingProgram);
    
    // 处理模具扩展数据
    if(program[0].action === Teach.actions.ACT_COMMENT){
        if(program[0].reserve !== undefined){
            PData.moldExtentData = program[0].reserve;
        }
    }

    // 遍历程序数组，构建模型
    for(var p = 0; p < program.length; ++p){
        step = program[p];
        
        // 设置插入索引
        step["insertedIndex"] = step.hasOwnProperty("insertedIndex") ? 
            step.insertedIndex : insertedIndex++;

        if(step.insertedIndex >= insertedIndex)
            insertedIndex = step.insertedIndex + 1;
            
        // 解析点位
        if(Teach.canActionUsePoint(step)){
            Teach.currentRecord.definedPoints.parseActionPoints(step);
        }
        
        // 处理标志动作
        if(step.action === Teach.actions.ACT_FLAG){
            at = Teach.actionTypes.kAT_Flag;
            Teach.currentRecord.flagsDefine.pushFlag(
                currentParsingProgram, 
                new Teach.FlagItem(step.flag, step.comment)
            );
        }
        // 处理同步开始
        else if(step.action === Teach.actions.F_CMD_SYNC_START){
            at = Teach.actionTypes.kAT_SyncStart;
            isSyncStart = true;
        }
        // 处理同步结束
        else if(step.action === Teach.actions.F_CMD_SYNC_END){
            at = Teach.actionTypes.kAT_SyncEnd;
            isSyncStart = false;
        }
        // 处理跳转动作
        else if(Teach.isJumpAction(step.action)){
            jumpLines.push(p);
            at = Teach.actionTypes.kAT_Flag;
        }
        // 处理等待动作
        else if(step.action === Teach.actions.F_CMD_IO_INPUT ||
                 step.action === Teach.actions.F_CMD_WATIT_VISION_DATA ||
                 step.action === Teach.actions.F_CMD_LONG_DISTANCE_COMMAND ||
                 step.action === Teach.actions.F_CMD_VISON_REMOTE_ACTION){
            at = Teach.actionTypes.kAT_Wait;
        }
        // 处理路径平滑开始
        else if(step.action === Teach.actions.F_CMD_PATH_SMOOTH_BEGIN){
            at = Teach.actionTypes.kAT_PSStart;
            isPSStart = true;
        }
        // 处理路径平滑结束
        else if(step.action === Teach.actions.F_CMD_PATH_SMOOTH_END){
            at = Teach.actionTypes.kAT_PSEnd;
            isPSStart = false;
        }
        else
            at = Teach.actionTypes.kAT_Normal;
            
        // 同步和路径平滑状态传递
        if(isSyncStart)
            at = Teach.actionTypes.kAT_SyncStart;
        if(isPSStart)
            at = Teach.actionTypes.kAT_PSStart;

        // 添加到模型
        model.append(new Teach.ProgramModelItem(step, at));
    }
    
    return insertedIndex;
}
```

**功能**：
1. 清空现有程序模型
2. 解析程序数组中的每个动作
3. 设置动作类型（普通、同步、标志、等待等）
4. 处理点位、标志、跳转等特殊动作
5. 将动作添加到 ListModel 中

### 3.5 步骤8：收集特殊行信息

```javascript
programInstance.collectSpecialLines(targetProgramIndex);
```

**功能**: 
- 收集计数器、定时器、堆垛、点位等特殊指令的行号信息
- 详见《通用程序指令说明.md》中的"特殊数据跟踪"章节

### 3.6 步骤9：整理编号

```javascript
programInstance.onFixIndexTriggered();
```

**文件**: `qml/App_HAMOUI/teach/ProgramFlowPage.qml`

```javascript
function onFixIndexTriggered(){
    var cM = currentModel();
    var ao;
    
    // 重新分配所有指令的 insertedIndex
    for(var i = 0; i < cM.count; ++i){
        ao = cM.get(i).mI_ActionObject;
        ao.insertedIndex = i;
        cM.setProperty(i, "mI_ActionObject", ao);
    }
    
    // 更新下一个插入索引
    PData.programToInsertIndex[currentProgramIndex()] = i;
}
```

**功能**: 
- 重新整理所有指令的 `insertedIndex`，确保连续性
- 从 0 开始递增，消除删除指令后的空隙

### 3.7 步骤10：标记为已修改

```javascript
programInstance.hasModify = true;
```

**说明**: 
- 设置 `hasModify` 标志为 `true`
- 这是保存的前提条件（`saveProgram` 函数会检查此标志）

### 3.8 步骤11：保存程序

```javascript
programInstance.saveProgram(targetProgramIndex);
```

**文件**: `qml/App_HAMOUI/teach/ProgramFlowPage.qml`

这个函数与通用编程界面使用的是同一个保存函数，详见《通用程序指令说明.md》中的"保存状态"章节。

**核心流程**：
1. 检查 `hasInit` 和 `hasModify` 标志
2. 调用 `beforeSaveProgram(which)` 前置处理
3. 显示"Program Compiling..."提示
4. 根据程序类型调用不同的保存方法：
   - 主程序：`panelRobotController.saveMainProgram(modelToProgram(0))`
   - 子程序：`panelRobotController.saveSubProgram(which, modelToProgram(which))`
5. 将 ListModel 转换为 JSON 字符串
6. 通过 C++ 控制器保存到数据库
7. 发送到主机：`sendMainProgramToHost()` 或 `sendSubProgramToHost(which)`
8. 处理错误信息
9. 记录操作日志
10. 设置 `hasModify = false`
11. 调用 `afterSaveProgram(which)` 后置处理

## 四、与通用编程界面的对比

### 4.1 相同点

| 特性 | 简单编程 | 通用编程 |
|-----|---------|---------|
| **保存函数** | 使用同一个 `saveProgram()` | 使用同一个 `saveProgram()` |
| **程序模型** | 使用同一个 `ProgramFlowData.programs` | 使用同一个 `PData.programs` |
| **持久化方式** | C++ 控制器保存到数据库 | C++ 控制器保存到数据库 |
| **数据格式** | JSON 字符串 | JSON 字符串 |
| **断电保持** | 支持 | 支持 |
| **特殊行跟踪** | 支持（计数器、定时器等） | 支持（计数器、定时器等） |

### 4.2 不同点

| 特性 | 简单编程 | 通用编程 |
|-----|---------|---------|
| **程序构建方式** | 从模板槽位构建 | 逐条插入指令 |
| **前置验证** | 检查槽位完整性 | 无特殊验证 |
| **程序来源** | `stepSlots` 数组 | `ListModel` 直接编辑 |
| **更新模型** | 先清空再重建 | 增量修改 |
| **整理编号** | 保存前自动整理 | 手动触发或保存时 |
| **用户界面** | 向导式，分步骤 | 自由编辑 |
| **适用场景** | 标准化工艺流程 | 灵活的自定义程序 |

## 九、总结

### 9.1 简单编程保存流程特点

1. **向导式验证**: 保存前进行多重验证，确保程序完整性
2. **模板驱动**: 从模板槽位构建程序，而非直接编辑 ListModel
3. **自动整理**: 保存前自动整理编号，确保索引连续
4. **部分持久化**: 只保存了程序指令，但槽位数据本身未持久化

### 9.2 断电保持机制分析

#### 9.2.1 保存的内容（有断电保持）

简单编程调用 `saveProgram()` 后，以下内容会被持久化：

1. **程序指令**: 从 `stepSlots` 构建的程序数组被保存到 `ProgramFlowData.programs[targetProgramIndex]`
2. **ListModel 数据**: 通过 `updateProgramModel()` 更新的 ListModel
3. **数据库存储**: 通过 `panelRobotController.saveMainProgram()` 或 `saveSubProgram()` 保存到数据库
4. **断电保持**: 程序指令可以断电保持

**持久化流程**：
```
stepSlots (槽位数组)
  ↓ 构建程序数组
program (动作对象数组)
  ↓ updateProgramModel
ProgramFlowData.programs[index] (ListModel)
  ↓ modelToProgram
JSON 字符串
  ↓ saveMainProgram/saveSubProgram
数据库 (断电保持)
```

#### 9.2.2 未保存的内容（无断电保持）

以下内容没有被持久化，断电后会丢失：

1. **槽位数据**: `stepSlots` 数组本身（包含 `stepIndex`、`actionObject`、`slotName` 等）
2. **模板状态**: 当前选择的模板信息
3. **编辑状态**: 当前编辑的步骤索引、功能面板状态等
4. **路径/逻辑分类**: `pathStepSlots` 和 `logicStepSlots` 数组

**问题根源**：
```javascript
property variant stepSlots: []  // 只是一个普通的 JavaScript 数组属性
property variant pathStepSlots: []
property variant logicStepSlots: []
property variant currentTemplateDefinition: null
```

这些属性只存在于 QML 组件的内存中，没有任何持久化机制。

### 9.3 为什么没有完全断电保持

#### 9.3.1 设计问题

简单编程界面的设计存在以下问题：

1. **双重数据结构**: 
   - `stepSlots`: 简单编程界面的槽位数据（未持久化）
   - `ProgramFlowData.programs`: 通用编程界面的程序数据（已持久化）

2. **单向转换**: 
   - 保存时：`stepSlots` → `program` → `ListModel` → 数据库 ✓
   - 加载时：数据库 → `ListModel` → `stepSlots` ✗（缺失）

3. **缺少反向加载**: 
   - 没有从 `ListModel` 或数据库恢复 `stepSlots` 的机制
   - 重启后，`stepSlots` 为空数组，无法恢复编辑状态

#### 9.3.2 实际影响

**场景1：正常保存后断电**
```
1. 用户在简单编程界面编辑程序
2. 点击"保存"按钮
3. 程序指令被保存到数据库 ✓
4. 断电重启
5. 程序指令可以在通用编程界面看到 ✓
6. 但简单编程界面的槽位数据丢失 ✗
7. 无法继续在简单编程界面编辑 ✗
```

**场景2：未保存就断电**
```
1. 用户在简单编程界面编辑程序
2. 未点击"保存"按钮
3. 断电重启
4. 所有数据丢失 ✗
```

### 9.4 关键差异总结

| 特性 | 简单编程 | 通用编程 |
|-----|---------|---------|
| **数据结构** | `stepSlots` 数组 + ListModel | ListModel |
| **持久化数据** | 仅 ListModel | ListModel |
| **断电保持** | 部分（仅程序指令） | 完全 |
| **编辑状态恢复** | 不支持 | 支持 |
| **数据一致性** | 双重结构，可能不一致 | 单一结构，一致 |

### 9.5 解决方案建议

要实现完全的断电保持，需要：

1. **持久化槽位数据**: 
   - 将 `stepSlots` 保存到数据库或配置文件
   - 可以使用 `panelRobotController.setCustomSettings()` 保存

2. **实现反向加载**: 
   - 从数据库加载槽位数据
   - 在 `Component.onCompleted` 或页面显示时恢复

3. **保存模板状态**: 
   - 保存当前选择的模板信息
   - 保存编辑状态（当前步骤索引等）

4. **统一数据结构**: 
   - 考虑直接使用 ListModel，避免双重数据结构
   - 或者确保 `stepSlots` 和 ListModel 的同步
