# 取片工艺教导插入流程

## 概述

本文档详细说明了在取片工艺（薄板工艺 ThinSectionProcess）中，从选择工艺类型到将取片模块参数插入教导程序的完整流程。

## 流程架构图

```
用户操作
  ↓
工艺类型选择 (ProductSettings.qml)
  ↓
教导程序页面 (ProgramPage.qml)
  ↓
程序流程页面 (ProgramFlowPage.qml)
  ↓
工艺模块菜单 (ProcessMenu.qml)
  ↓
取片工艺模块 (ThinSectionCraft.qml)
  ↓
生成指令对象序列 (createActionObjects)
  ↓
批量插入教导程序 (insertActionToList)
```

### 5. 取片工艺模块（核心）

**文件**: `qml/App_HAMOUI/teach/extents/ThinSectionCraft.qml`

这是生成取片指令序列的核心模块，会生成一整套完整的取片流程指令（通常有几十条指令）：

#### 主要功能

1. **参数收集**：收集用户配置的所有取片参数
2. **计数器创建**：创建取物总数和已取堆数计数器
3. **标志位创建**：创建初始化、探料、成功/失败等标志位
4. **指令序列生成**：生成完整的取片流程指令

#### 核心函数

```qml
ExtentActionEditorBase {
    id: thinSectionContainer
    
    function createActionObjects() {
        var ret = [];
        
        // 1. 收集参数
        var dataToSave = {
            "plane": planeSel.configValue,              // 取片平面（0=XY, 1=XZ, 2=YZ）
            "seq": seq.configValue,                     // 取片顺序（0=先行后列, 1=先列后行）
            "count1": parseFloat(count1.configValue),   // 第一方向堆数
            "count2": parseFloat(count2.configValue),   // 第二方向堆数
            "space1": parseFloat(space1.configValue),   // 第一方向间距（mm）
            "space2": parseFloat(space2.configValue),   // 第二方向间距（mm）
            "expLen": parseFloat(expLen.configValue),   // 探料长度（mm）
            "maxExpLen": parseFloat(maxExpLen.configValue), // 最大探料长度（mm）
            "startPos": {...},                          // 起始点位（6轴坐标）
            "startPosSpeed": parseFloat(startPosSpeed.configValue),   // 起点速度（%）
            "fastExpSpeed": parseFloat(fastExpSpeed.configValue),     // 快速探料速度（%）
            "slowExpSpeed": parseFloat(slowExpSpeed.configValue),     // 慢速探料速度（%）
            "desktopOffset": parseFloat(desktopOffset.configValue),   // 台面高度差（mm）
            "output": outputActionEditor.outputActionDefines(),       // 输出动作（治具）
            "input": inputActionEditor.inputActionDefines(),          // 输入检测（停止信号）
            "openFixtureBeforeExp": openFixtureBeforeExp.isChecked,  // 探料前开治具
            "changeStackWhenExpFail": changeStackWhenExpFail.isChecked, // 探料失败换堆
            "thinGroup": thinGroup.currentIndex                       // 取片组号（0-3）
        };
        
        var totalStack = dataToSave.count1 * dataToSave.count2;  // 总堆数
        
        // 2. 生成注释行（第一行，包含所有参数）
        ret.push(Teach.generateCommentAction(dataShowInComment));
        ret[0].innerData = JSON.stringify(dataToSave);
        
        // 3. 创建计数器
        // 计数器 thinGroupVal*2+0：取物总数（纯计数，无目标）
        // 计数器 thinGroupVal*2+1：已取堆数（目标计数，目标=totalStack）
        
        // 4. 创建标志位
        // - Init Begin/End：初始化开始/结束
        // - Got All：取完所有堆
        // - Get Finished：单次取片完成
        // - Fast Exp：快速探料
        // - Slow Exp：慢速探料
        // - Exp Success：探料成功
        // - Exp Fail：探料失败
        // - Stack-N StartPos：第N堆起点（N=1~totalStack）
        
        // 5. 生成初始化指令（检查是否首次运行）
        
        // 6. 生成定位起点指令（为每个堆生成一个起点标志和移动指令）
        
        // 7. 生成探料指令（快速探料 + 慢速探料）
        
        // 8. 生成探料成功处理（保存位置、开治具、计数）
        
        // 9. 生成探料失败处理（重新探料或换堆）
        
        // 10. 生成取完报警
        
        // 11. 生成模块结束指令
        
        return ret;  // 返回完整指令序列（通常30-50条指令）
    }
}
```

function onInsertTriggered() {
    // 调用动作编辑器框架的createActionObjects()
    // 这会调用到ProcessMenu.createActionObjects()
    // 最终调用到ThinSectionCraft.createActionObjects()
    var actionObjects = actionEditorFrame.item.createActionObjects();
    
    // 检查是否有有效的指令对象
    if(actionObjects == "undefined" || actionObjects.length == 0) return;
    
    // 显示确认提示框（如果启用）
    if(!tipBox.showTip) {
        tipBox.showCheck(
            qsTr("Make sure to insert program?"),
            qsTr("Yes"),
            qsTr("No"),
            qsTr("Do Not Show Again"),
            tipBox.showTip
        );
        tipBox.checkFinished.connect(tipBox.showProgramTip);
    } else {
        // 直接插入程序
        _InsertPrograms(actionObjects);
    }
}
```

    for(var i = 0; i < obj.length; ++i) {
        insertActionToList(obj[i]);
    }
    
    // 刷新程序列表显示
    repaintProgramItem(currentModel());
    
    // 定位到当前行
    programListView.positionViewAtIndex(programListView.currentIndex, ListView.Visible);
}

function insertActionToList(actionObject) {
    var cI = programListView.currentIndex;  // 当前行索引
    if(cI < 0) return;
    
    // 设置插入索引
    actionObject.insertedIndex = PData.programToInsertIndex[currentProgramIndex()]++;
    
    var cPI = currentProgramIndex();
    var model = currentModel();
    
    // 同步特殊行信息（在插入位置之后的所有行号+1）
    PData.counterLinesInfo.syncLines(cPI, cI, 1);  // 计数器行
    PData.timerLinesInfo.syncLines(cPI, cI, 1);    // 定时器行
    PData.stackLinesInfo.syncLines(cPI, cI, 1);    // 堆栈行
    PData.pointLinesInfo.syncLines(cPI, cI, 1);    // 点位行
    
    // 处理标志定义
    if(actionObject.action === Teach.actions.ACT_FLAG) {
        Teach.currentRecord.flagsDefine.pushFlag(cPI, new Teach.FlagItem(...));
    }
    
    // 处理计数器ID
    if(Teach.hasCounterIDAction(actionObject)) {
        var cs = Teach.actionCounterIDs(actionObject);
        for(var c in cs) {
            PData.counterLinesInfo.add(cPI, cs[c], cI);
        }
    }
    
    // 处理定时器ID、堆栈ID、点位引用...
    
    // 插入到模型中
    model.insert(cI++, new Teach.ProgramModelItem(actionObject, Teach.actionTypes.kAT_Normal));
    
    // 标记程序已修改
    hasModify = true;
    
    // 记录操作日志
    ICOperationLog.opLog.appendOperationLog(text);
    
    // 保存到撤销缓冲区
    revokebuff.push({...});
}
```

### 计数器说明

取片工艺使用两个计数器（每个取片组使用2个）：

| 计数器ID | 名称 | 类型 | 说明 |
|---------|------|------|------|
| thinGroup*2+0 | 取物总数 | 纯计数 | 记录总共取了多少片，无目标值 |
| thinGroup*2+1 | 已取堆数 | 目标计数 | 记录已取了多少堆，目标值=count1×count2 |

例如：取片组1（thinGroup=0）使用计数器0和1，取片组2使用计数器2和3。

### 内存地址使用

取片工艺使用16个连续的内存地址（每个取片组使用16个）：

| 地址偏移 | 用途 | 说明 |
|---------|------|------|
| 800-802 | 探料位置 | 保存探料成功后的位置（X/Y/Z或对应轴） |
| 803-805 | 保留 | 预留地址 |
| 808 | 堆位置 | 保存当前堆的起点位置 |
| 其他 | 保留 | 预留地址 |

实际地址 = thinGroup * 16 + 偏移

例如：取片组1（thinGroup=0）使用地址800-815，取片组2使用地址816-831。

## 完整示例

### 用户操作步骤

1. **选择工艺类型**
   - 进入产品设置页面
   - 在"Technology type"下拉框中选择"ThineSectionProcess"（取片工艺）
   - 系统提示需要断电重启

2. **进入教导程序页面**
   - 切换到教导程序页面
   - 选择要编辑的程序（通常是模块程序）
   - 选择要插入的位置（在列表中选中某一行）

3. **打开工艺模块**
   - 点击"Editor S/H"按钮打开动作编辑器
   - 选择"工艺"菜单
   - 系统自动加载取片工艺模块

4. **设置取片参数**
   - **取片组**：选择G1-G4（用于多组取片配置）
   - **取片平面**：选择XY/XZ/YZ
   - **取片顺序**：选择先行后列/先列后行
   - **堆数配置**：设置count1和count2（如3×2=6堆）
   - **间距配置**：设置space1和space2（mm）
   - **起始点位**：点击"Set In"示教当前位置，或手动输入6轴坐标
   - **探料参数**：
     - 探料长度（expLen）：正值向下，负值向上
     - 最大探料长度（maxExpLen）：防止探料过深
     - 起点速度（startPosSpeed）：移动到起点的速度
     - 快速探料速度（fastExpSpeed）：快速探料阶段的速度
     - 慢速探料速度（slowExpSpeed）：慢速探料阶段的速度
   - **台面高度差**：补偿台面不平整
   - **输出配置**：点击"Output"按钮配置治具输出
   - **输入配置**：点击"Input"按钮配置停止信号检测
   - **选项**：
     - 探料前开治具：勾选则在探料前开治具
     - 探料失败换堆：勾选则探料失败后自动换堆

5. **保存为模块**（可选）
   - 点击"Save"按钮
   - 输入模块名称（如"G1-ThinSec-XY-3X2"）
   - 保存后可以通过"Read Module Info"读取

   - 点击"Save"按钮保存修改

## 注意事项

1. **工艺类型切换**
   - 切换工艺类型后必须断电重启才能完全生效
   - 切换前建议保存当前程序

2. **参数合法性**
   - 堆数：count1和count2必须大于0
   - 间距：space1和space2可以为负值（反向排列）
   - 探料长度：expLen可以为负值（向上探料）
   - 速度范围：0-100%

3. **内存地址冲突**
   - 不同取片组使用不同的内存地址段，互不干扰
   - G1使用800-815，G2使用816-831，G3使用832-847，G4使用848-863

4. **计数器冲突**
   - 不同取片组使用不同的计数器，互不干扰
   - G1使用C0和C1，G2使用C2和C3，G3使用C4和C5，G4使用C6和C7

5. **模块程序**
   - 取片指令通常保存为模块程序，方便在主程序中调用
   - 模块名称建议包含取片组、平面、堆数等信息，如"G1-ThinSec-XY-3X2"

6. **探料失败处理**
   - 如果勾选"探料失败换堆"，探料失败后会自动换到下一堆
   - 如果不勾选，探料失败后会重新探料，直到超过最大探料长度

7. **程序修改标志**
   - 插入指令后 `hasModify` 标志会被设置为 `true`
   - 退出前需要保存程序，否则修改会丢失

8. **撤销功能**
   - 每次插入操作都会保存到撤销缓冲区
   - 可以通过撤销功能恢复到插入前的状态

# 与通用指令插入流程的对比

## 核心机制对比

取片工艺插入流程与通用程序指令插入流程在**底层机制上完全一致**，但在**应用层面有显著差异**。

### 相同点（核心流程一致）

**1. 插入触发机制相同**

两种流程都使用相同的触发链：

```
用户点击"插入"按钮
  ↓
ProgramFlowPage.onInsertTriggered()
  ↓
actionEditorFrame.item.createActionObjects()  // 获取指令对象数组
  ↓
_InsertPrograms(actionObjects)  // 批量插入
  ↓
insertActionToList(actionObject)  // 逐个插入
```

**2. 核心插入函数相同**

都使用 `insertActionToList()` 函数，执行相同的操作：
- 同步特殊行信息（计数器、定时器、堆垛、点位）
- 处理标志位、计数器等特殊动作
- 插入到 ListModel
- 标记 `hasModify = true`
- 记录操作日志
- 保存到撤销缓冲区

**3. 数据存储位置相同**

- **未保存时**：都存储在内存中的 `PData.programs` ListModel
- **保存后**：都通过 `panelRobotController.saveMainProgram()` 或 `saveSubProgram()` 持久化到模具数据库

### 主要区别（应用层面）

**1. 指令来源不同**

| 对比项 | 通用指令插入 | 取片工艺插入 |
|-------|------------|------------|
| **指令来源** | 通用动作编辑器 | 取片工艺模块（ThinSectionCraft.qml） |
| **指令数量** | 通常1条或少量 | 一次生成30-70条指令 |
| **生成方式** | 用户手动配置单个动作 | 根据参数自动生成完整流程 |
| **用户操作** | 逐个配置每条指令 | 配置一次参数，自动生成整套流程 |

**2. createActionObjects() 实现不同**

**通用流程**：
```javascript
// 从动作编辑器获取，可能是单个动作
actionEditorFrame.item.createActionObjects()
  ↓
返回 [actionObject]  // 通常1条指令
```

**取片工艺流程**：
```javascript
// ProcessMenu.qml 调用
ProcessMenu.createActionObjects()
  ↓
// 转发到取片工艺模块
ThinSectionCraft.createActionObjects()
  ├─ 收集参数（平面、堆数、间距等）
  ├─ 创建计数器（取物总数、已取堆数）
  ├─ 创建标志位（初始化、探料成功/失败等）
  └─ 生成指令序列
  ↓
返回 [
  注释行（包含参数）,
  计数器跳转,
  内存比较跳转,
  标志位,
  定位起点指令（循环N次）,
  探料指令,
  探料成功处理,
  探料失败处理,
  取完报警,
  模块结束
]  // 30-70条指令
```

**3. 插入规模不同**

**通用流程**：
```javascript
// 通常插入1条或几条指令
_InsertPrograms([actionObject1, actionObject2])
  ↓
insertActionToList() 调用 1-3 次
```

**取片工艺流程**：
```javascript
// 一次插入30-70条指令
_InsertPrograms([
  actionObject1,   // 注释行
  actionObject2,   // 计数器跳转
  actionObject3,   // 内存比较
  ...
  actionObject50,  // 模块结束
])
  ↓
insertActionToList() 调用 30-70 次
```

**4. 特殊处理不同**

**取片工艺的特殊之处**：

1. **自动创建资源**：
   - 自动创建计数器（取物总数、已取堆数）
   - 自动创建多个标志位（初始化、探料成功/失败等）
   - 自动分配内存地址（800-815等）

2. **循环生成指令**：
   ```javascript
   // 为每个堆生成一组指令
   for(var i = 0; i < totalStack; i++) {
       ret.push(标志：堆N起点);
       ret.push(直线移动到堆N);
       ret.push(保存位置);
       ret.push(跳转到探料);
   }
   ```

3. **参数序列化**：
   ```javascript
   // 第一条注释行包含所有参数
   ret[0].innerData = JSON.stringify(dataToSave);
   ```

4. **智能计算**：
   - 根据平面（XY/XZ/YZ）自动计算每个堆的坐标
   - 根据顺序（先行后列/先列后行）自动排列堆位置
   - 根据间距自动计算偏移量

## 流程对比图

**通用指令插入流程**：
```
用户配置单个动作
  ↓
createActionObjects() → [1个actionObject]
  ↓
_InsertPrograms([actionObject])
  ↓
insertActionToList(actionObject) × 1次
  ↓
插入1条指令到ListModel
  ↓
hasModify = true
```

**取片工艺插入流程**：
```
用户配置取片参数（平面、堆数、间距等）
  ↓
ThinSectionCraft.createActionObjects()
  ├─ 收集参数
  ├─ 创建计数器
  ├─ 创建标志位
  └─ 生成指令序列 → [30-70个actionObjects]
  ↓
_InsertPrograms([actionObject1, ..., actionObject50])
  ↓
for i = 0 to 49:
  insertActionToList(actionObject[i]) × 50次
  ↓
插入50条指令到ListModel
  ↓
hasModify = true
```

## 架构设计优势

这种设计体现了良好的软件架构原则：

1. **底层统一**：
   - 所有指令插入都走相同的核心流程（`insertActionToList`）
   - 统一的数据存储机制（ListModel → 数据库）
   - 统一的撤销/重做机制

2. **上层灵活**：
   - 可以是手动单条插入（通用动作编辑器）
   - 也可以是自动批量生成（取片工艺模块）
   - 不同工艺模块可以有不同的生成策略

3. **易于扩展**：
   - 新增其他工艺（如喷涂、冲压、压铸）也可以复用相同的插入机制
   - 只需实现各自的 `createActionObjects()` 函数
   - 底层的 `insertActionToList()` 无需修改

4. **职责分离**：
   - **上层**：负责指令生成逻辑（工艺特定）
   - **中层**：负责批量插入协调（`_InsertPrograms`）
   - **底层**：负责单条指令插入（`insertActionToList`）
   - **存储层**：负责数据持久化（`saveMainProgram`/`saveSubProgram`）

## 对比总结表

| 特性 | 通用指令插入 | 取片工艺插入 |
|-----|------------|------------|
| **核心机制** | insertActionToList() | insertActionToList()（相同） |
| **数据存储** | ListModel → 数据库 | ListModel → 数据库（相同） |
| **指令数量** | 1-3条 | 30-70条 |
| **生成方式** | 手动配置 | 自动生成 |
| **资源管理** | 手动创建 | 自动创建（计数器、标志位、内存） |
| **参数配置** | 逐条配置 | 一次配置，批量生成 |
| **适用场景** | 简单动作、调试 | 复杂工艺流程 |
| **用户体验** | 灵活但繁琐 | 高效但固定 |

## 结论

**取片工艺插入流程是通用程序指令插入流程的高级应用**：

- **底层使用相同的 `insertActionToList()` 机制**
- **上层通过智能生成器（ThinSectionCraft）批量创建指令**
- **最终都存储在相同的 ListModel 中，保存到相同的数据库**

这种设计使得系统既保持了底层的一致性和稳定性，又提供了上层的灵活性和扩展性，是一个优秀的分层架构设计案例。

## 总结

取片工艺教导插入流程的特点：

1. **一次性生成完整流程**：不同于单条指令插入，取片工艺一次生成几十条指令，形成完整的取片流程
2. **自动化程度高**：自动创建计数器、标志位，自动计算每个堆的位置
3. **灵活配置**：支持多种取片平面、顺序、探料策略
4. **多组支持**：支持4个独立的取片组（G1-G4），互不干扰
5. **容错处理**：包含探料失败重试、换堆、取完报警等容错机制
6. **复用底层机制**：与通用指令插入共享相同的核心插入逻辑，保证了系统的一致性

整个流程通过 QML 的信号槽机制和 JavaScript 函数调用实现了松耦合的模块化设计，便于维护和扩展。
