# 对tryP0进行详细分析

tryP0 (line 1456) 在这个文件里不是普通显示框，它本质上是“P0 点试跑按钮 + 点位信息显示框”。

它在 PointEdit.qml 里的用法可以分成几部分看：

1. 基本角色
在 PointEdit.qml (line 1456) 这里它是一个 ICButton：
- id: tryP0
- bgColor: "yellow"
- height: 80
- width: selP0ReferenceName.width
- enabled: points[0] != null

意思是：
- 它显示的是 P0 当前点的信息
- 只有 points[0] 已经存在时才能按

2. 按下和松开的行为
在 PointEdit.qml (line 1462)：
- onBtnPressed 调 tryRunPoint(points[0]) (line 550)
- onBtnReleased 调 panelRobotController.sendKnobCommandToHost(Keymap.CMD_MANUAL_STOP)

这说明它不是“点击一次执行”，而是：
- 按住时试跑到这个点
- 松开时停止
- 所以它更像“按住试运行”按钮。

3. 它显示什么文字
tryP0.text 不是写死的，而是在文件里多处动态更新。
核心更新函数是 showPointText() (line 1377)：
- 普通路径时：
    - tryP0.text = pointToShow(points[0], qsTr("Click to try target"))
- 曲线/圆弧时：
    - 可能变成 Click to try pass
    - 或 Click to try Center

也就是说：
- 没点位时显示提示文字
- 有点位时通过 pointToShow() (line 568) 把点位内容格式化显示出来

4. 什么时候刷新它
最重要的是 onPointsChanged (line 748)：
    当 points 变化后，最后会调 rightTopContaner.showPointText() (line 798)，进而刷新 tryP0.text
所以 tryP0 显示内容不是自己算的，而是跟 points[0] 绑定在整套点位编辑流程里。

5. 它和另外两个按钮的关系
tryP0 和这两个按钮是一组：
- setAsP0 (line 1424)
- manualInputP0 (line 1436)

流程是：
- setAsP0 采当前机器人位置写入 points[0]
- manualInputP0 打开手动输入界面编辑 points[0]
- tryP0 用当前 points[0] 做试跑显示和按住运行
- 所以 tryP0 依赖 points[0]，不是独立控件。

6. 它真正依赖的上下文
这个控件在本文件里依赖很多上下文，至少包括：
- points (line 27)
- tryRunPoint() (line 550)
- pointToShow() (line 568)
- showPointText() (line 1377)
- panelRobotController
- Keymap

所以它虽然看起来只是一个黄色按钮，实际上在这个文件里承担了：
- 点位文本显示
- 点位是否有效的状态表达
- 按住试跑
- 松开停止

    你希望在"简单编程"的"模板快设"页面点进"路径"功能时，底栏能显示实时坐标信息，而不是固定的提示文本。

2. 初步方案（错误）
    最开始我误解了需求，以为是在固定提示文本中动态显示坐标字符串：
    ```C++
    text: qsTr("当前坐标：X=0.00 Y=0.00 Z=0.00 U=0.00 V=0.00 W=0.00")
    ```
    这个方案是错误的，因为坐标是固定的 "0.00"。

3. 理解真实需求
    经过沟通，我理解到：
    - 默认情况：显示固定提示文本
    - 点进路径功能：应该使用和 ProgramPage 一样的 AxisPosDisplayBar 组件显示实时坐标

4. 实现方案
    步骤1：添加信号传递机制
    在 EasyProgramQuickPage.qml 中：
    ```qml
    signal functionPanelChanged(int panelIndex)

    function openFunctionPanel(index) {
        currentFunctionIndex = index;
        functionPanelChanged(index);  // 发出信号
    }

    function closeFunctionPanel() {
        currentFunctionIndex = -1;
        functionPanelChanged(-1);  // 发出信号
    }
    ```
    步骤2：在父组件监听信号
    在 EasyProgramWizard.qml 中：
    ```C++
    property int currentFunctionPanel: -1  // 跟踪当前打开的功能面板

    EasyProgramQuickPage {
        id: quickPage
        onFunctionPanelChanged: {
            currentFunctionPanel = panelIndex;  // 更新状态
        }
    }
    ```
    步骤3：创建两个底栏组件
    ```C++
    // 固定提示文本
    Item {
        id: hintBar
        visible: currentFunctionPanel !== 0
        Text {
            text: qsTr("请进入具体的设定页面进行设置，点击右下方的返回按钮返回上一层菜单")
        }
    }

    // 实时坐标显示
    AxisPosDisplayBar {
        id: posDisplayBar
        visible: currentFunctionPanel === 0
    }
    ```
    步骤4：动态切换底栏
    ```C++
    statusSection: currentFunctionPanel === 0 ? posDisplayBar : hintBar
    ```
    步骤5：处理页面切换时关闭功能面板
    ```C++
    onMenuItem1Triggered: {  // 上一页
        if (currentPageIndex === 4 && currentFunctionPanel >= 0) {
            quickPage.closeFunctionPanel();  // 关闭功能面板
        }
        currentPageIndex--;
    }
    ```

5. 遇到的问题和修复
    问题1: 界面顶部出现浅色坐标文字
    - 原因: AxisPosDisplayBar 没有设置 visible 属性，一直在渲染
    - 解决: 添加 visible: currentFunctionPanel === 0
    问题2: 调用不存在的方法 startTimer()
    - 原因: AxisPosDisplayBar 没有这个方法
    - 解决: 移除 onVisibleChanged 中的方法调用

6. 最终实现效果
    - 默认状态: 显示固定提示文本
    - 点进路径: 自动切换到 AxisPosDisplayBar，显示实时坐标
    - 返回或切换功能: 自动切换回固定提示文本
    - 页面切换: 自动关闭功能面板，下次需要重新点击

7. 核心技术点
    - 信号机制: 使用 QML 的 signal 在子组件和父组件间通信
    - 动态属性绑定: statusSection 根据状态动态切换组件
    - 可见性控制: 使用 visible 属性控制组件显隐
    - 状态管理: 使用 currentFunctionPanel 跟踪当前状态
    这个方案实现了底栏的动态切换，既保持了简单编程的固定提示，又能在需要时显示实时坐标信息！

# 为什么在模板快设页面会出现具体是哪个模板
现在在 EasyProgramWizard.qml 里把“第 4 页选中的模板名”传给了第 5 页，而第 5 页又把它显示出来了。

链路是这样的：
1. 第 4 页点“载入”时，触发 onLoadRequested
    在 EasyProgramWizard.qml (line 96)：
    ```qml
    onLoadRequested: {
        selectedTemplateKey = templateKey;
        selectedTemplateName = templateName;
        quickPage.templateKey = templateKey;
        quickPage.templateName = templateName;
        currentPageIndex = 4;
        menuItemTexts = getMenuItemTexts();
    }
    ```
    这里做了两件事：
    - 保存到了向导自己的 selectedTemplateName
    - 还直接赋给了 quickPage.templateName
2. 第 5 页又接收了这个值
    在 EasyProgramWizard.qml (line 112)：
    ```qml
    templateKey: root.selectedTemplateKey
    templateName: root.selectedTemplateName
    ```
    也就是说，EasyProgramQuickPage.qml 会拿到当前模板名。
3. 第 5 页把它画出来了
    我之前在 EasyProgramQuickPage.qml 标题栏里加了一个 Text，内容就是 templateName，所以你会看到“当前具体是哪个模板”。

总结：之所以会显示“具体是哪个模板”，是因为第 4 页的模板名被传给了第 5 页，而第 5 页又主动把 templateName 显示在标题栏里。


## 对于showTip的用法说明

showTip 的用法很简单，它是你这个页面里封装的一个提示函数：
```qml
function showTip(message, detail) {
    tipDialog.warning(message, qsTr("OK"), detail || "");
}
```
1. 常见调用方式：
    - showTip(qsTr("程序页面尚未就绪"))
    - showTip(qsTr("槽位未全部插入指令"), missingNames.join("\n"))
2. 参数含义：
    - message：主提示文本
    - detail：可选详情文本（不传时用空串）

## 为什么这个missingNames.join("\n")就能显示有哪些缺失

因为这条数据链正好是通的：
1. missingNames 里装的是每个缺失槽位的标题，比如 ["1.模外待机", "3.进入模区"]。
2. join("\n") 会把数组拼成一个字符串：
    "1.模外待机\n3.进入模区"
3. 这个字符串被当作 showTip 的 detail 传进去，最终到 ICMessageBox.warning(..., details)。
4. warning 里会执行 detailText.text = details，而 QML Text 会把 \n 当换行显示。

所以它“能显示有哪些”，本质是：先把“有哪些”放进数组，再用换行拼成一段详情文本显示出来。

# EasyProgramQuickPage.qml函数解释
1. cloneObject（72）深拷贝对象，避免直接改原数据。
2. initializeStepSlots（76）按 pathStepLayout/logicStepLayout 生成初始槽位数据。
3. refreshStepSlotViews（111）把总槽位拆成路径槽位和逻辑槽位两组供 UI 显示。
4. slotIndexByStep（125）根据 stepIndex 找槽位在数组里的下标。
5. currentStepSlot（134）取当前选中步骤对应的槽位对象。
6. buildStepComment（142）把标题里的“数字前缀”去掉，生成注释文本。
7. updateSlot（149）按 stepIndex 更新槽位字段，并触发视图刷新。
8. openFunctionPanel（166）打开某个功能面板并记录当前面板索引。
9. closeFunctionPanel（171）关闭功能面板（索引设为 -1）。
10. syncPathCustomNameForCurrentSlot（176）在路径面板下，同步未填充槽位的路径自定义名为空。
11. canPanelInsertToSlot（187）校验“当前面板”是否允许插入到“当前槽位”。
12. doInsertCurrentStepAction（206）真正执行插入：从面板生成 actionObject，写入槽位。
13. onInsertConfirmFinished（247）处理“插入确认框”回调。
14. insertCurrentStepAction（256）插入入口：先校验，再决定是否弹确认框。
15. doDeleteCurrentStepAction（284）真正执行删除：清空当前槽位动作数据。
16. editFilledStepAction（298）编辑已填充槽位：回到对应面板并回填动作参数。
17. onDeleteConfirmFinished（326）处理“删除确认框”回调。
18. deleteCurrentStepAction（335）删除入口：判断是否确认后删除。
19. missingSlots（355）收集所有还没插入指令的槽位。
20. fixIndex（365）触发程序索引修复（调用 programInstance.onFixIndexTriggered()）。
21. save（373）保存流程：检查槽位完整性 -> 组装程序 -> 更新模型 -> 修索引 -> 保存。
22. handleOperation（417）底部操作按钮分发（目前实现了“插入/删除”）。
