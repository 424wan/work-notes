# 程序指令存储流程说明

## 概述

本文档详细说明了机器人控制系统中，程序编辑界面（中间区域）的指令插入和保存的完整存储流程。

---

## 一、未保存状态（编辑中）

### 1.1 存储位置

当用户在编程界面插入指令但**未点击保存**时，数据存储在：

- **内存中的 QML ListModel**
- 具体路径：`PData.programs` 数组
  - 主程序：`PData.programs[0]`
  - 子程序1-16：`PData.programs[1]` ~ `PData.programs[16]`
  - 功能模块：`PData.kFunctionProgramIndex` (索引17)
  - 手动程序：`PData.kManualProgramIndex` (索引18)

### 1.2 数据结构

每个程序模型（ListModel）包含多个动作对象，每个动作对象包含：

```javascript
{
    mI_ActionObject: {
        action: 动作类型编号,
        insertedIndex: 插入索引,
        // 其他动作相关参数...
    }
}
```

### 1.3 插入操作流程

**文件位置**：`qml/App_HAMOUI/teach/ProgramFlowPage.qml`

**关键函数**：`onInsertTriggered()` 和 `insertActionToList()`

#### 主插入函数 `onInsertTriggered()`

```javascript
function onInsertTriggered(){
    // 从动作编辑器获取要插入的动作对象数组
    var actionObjects = actionEditorFrame.item.createActionObjects();
    
    // 检查是否有有效的动作对象
    if(actionObjects == "undefined" || actionObjects.length == 0) return;
    
    // 如果未设置"不再显示"，则显示确认对话框
    if(!tipBox.showTip){
        if(tipBox.visible && (tipBox.text == qsTr("Make sure to insert program?"))) return;
        tipBox.showCheck(qsTr("Make sure to insert program?"),
                        qsTr("Yes"),
                        qsTr("No"),
                        qsTr("Do Not Show Again"),
                        tipBox.showTip);
        tipBox.checkFinished.connect(tipBox.showProgramTip);
    }else{
        // 直接插入程序
        _InsertPrograms(actionObjects);
    }
}
```

#### 批量插入函数 `_InsertPrograms()`

```javascript
function _InsertPrograms(obj){
    // 遍历所有要插入的动作对象
    for(var i = 0; i < obj.length; ++i){
        insertActionToList(obj[i]);
    }
    // 重绘程序列表
    repaintProgramItem(currentModel());
    // 定位到当前行
    programListView.positionViewAtIndex(programListView.currentIndex, ListView.Visible);
}
```

#### 核心插入函数 `insertActionToList()`

```javascript
function insertActionToList(actionObject){
    var copyrevokebuff = [];
    var id;
    
    var cI = programListView.currentIndex;  // 获取当前选中行
    var oCI = cI;
    if(cI < 0) return;
    
    // 设置插入索引
    actionObject.insertedIndex = PData.programToInsertIndex[currentProgramIndex()]++;
    var cPI = currentProgramIndex();
    var model = currentModel();
    
    // 同步所有特殊行信息（从插入位置开始，所有行号+1）
    PData.counterLinesInfo.syncLines(cPI, oCI, 1);
    PData.timerLinesInfo.syncLines(cPI, oCI, 1);
    PData.stackLinesInfo.syncLines(cPI, oCI, 1);
    PData.pointLinesInfo.syncLines(cPI, oCI, 1);
    
    // 如果是标志动作，添加到标志定义中
    if(actionObject.action === Teach.actions.ACT_FLAG){
        Teach.currentRecord.flagsDefine.pushFlag(cPI, 
            new Teach.FlagItem(actionObject.flag, actionObject.comment));
    }
    
    // 如果动作包含计数器ID，记录行号
    if(Teach.hasCounterIDAction(actionObject)){
        var cs = Teach.actionCounterIDs(actionObject);
        for(var c in cs){
            PData.counterLinesInfo.add(cPI, cs[c], cI);
            id = cs[c];
        }
    }
    
    // 如果动作包含定时器ID，记录行号
    if(Teach.hasTimerIDAction(actionObject)){
        cs = Teach.actionTimerIDs(actionObject);
        for(c in cs){
            PData.timerLinesInfo.add(cPI, cs[c], cI);
            id = cs[c];
        }
    }
    
    // 如果动作包含堆垛ID，记录行号
    if(Teach.hasStackIDAction(actionObject)){
        PData.stackLinesInfo.add(cPI, Teach.actionStackID(actionObject), cI);
        id = Teach.actionStackID(actionObject);
    }
    
    // 如果动作使用点位，记录所有点位的行号
    if(Teach.canActionUsePoint(actionObject)){
        var points = Teach.currentRecord.definedPoints.parseActionPointsHelper(actionObject);
        for(var p = 0; p < points.length; ++p){
            PData.pointLinesInfo.add(cPI, points[p].index, cI);
            id = points[p].index;
        }
    }
    
    // 插入到内存中的ListModel
    model.insert(cI++, new Teach.ProgramModelItem(actionObject, Teach.actionTypes.kAT_Normal));
    
    hasModify = true;  // 标记为已修改（未保存）
    
    // 记录操作日志
    var text = ICString.icStrformat(
        qsTr("OP:Insert  Edit:{0},Model:{1},Line:{2}Log:{3}"),
        editing.currentText(),
        moduleSel.currentText(),
        (programListView.currentIndex-1)+"\n",
        actionObjectLogToText(actionObject)
    );
    ICOperationLog.opLog.appendOperationLog(text);
    
    // 保存到撤销缓冲区
    copyrevokebuff = revokebuff;
    copyrevokebuff.push({
        "programIndex": cPI,
        "edit": editing.currentText(),
        "currentmodule": moduleSel.currentText(),
        "id": id,
        "line": oCI,
        "actionObject": actionObject,
        "log": text
    });
    revokebuff = copyrevokebuff;
}
```

### 1.4 修改标记

系统使用 `hasModify` 标志来跟踪程序是否有未保存的修改：

- `hasModify = true`：有未保存的修改
- `hasModify = false`：所有修改已保存

## 二、保存状态（点击保存后）

### 2.1 保存触发

用户点击"保存"按钮时，触发保存流程：

**触发函数**：`onSaveTriggered()`

```javascript
function onSaveTriggered(){
    hasModify = true;
    saveProgram(currentProgramIndex());  // 调用保存函数
    hasModuleChanged = false;
}
```

### 2.2 保存流程详解

**核心函数**：`saveProgram(which)`

**文件位置**：`qml/App_HAMOUI/teach/ProgramFlowPage.qml` (约1126行)

#### 步骤1：前置处理

```javascript
function saveProgram(which){
    if(!hasInit) return;      // 未初始化则返回
    if(!hasModify) return;    // 无修改则返回
    
    beforeSaveProgram(which); // 保存前处理
    
    tipBox.runningTip(qsTr("Program Compiling..."));  // 显示编译提示
}
```

#### 步骤2：数据转换

将内存中的 ListModel 转换为 JSON 格式：

```javascript
function modelToProgram(which){
    var model = PData.programs[which];
    var ret = [];
    
    for(var i = 0; i < model.count; ++i){
        var obj = model.get(i).mI_ActionObject;
        ret.push(obj);  // 将每个动作对象添加到数组
    }
    
    return JSON.stringify(ret);  // 转换为JSON字符串
}
```

#### 步骤3：程序检查与保存

根据不同的程序类型，调用不同的保存方法：

### 2.3 最终存储位置

数据通过 C++ 的 `panelRobotController` 控制器保存到：

- **当前模具（Record）的数据库或配置文件**
- 存储路径由 `panelRobotController.currentRecordName()` 确定
- 数据格式：JSON 字符串

**C++ 控制器方法**：
- `saveMainProgram(jsonString)` - 保存主程序
- `saveSubProgram(index, jsonString)` - 保存子程序
- `setCustomSettings(key, value)` - 保存自定义设置

## 三、特殊数据跟踪

### 3.1 特殊行信息收集

系统会跟踪特殊类型的指令行号，包括：

- **计数器**：`PData.counterLinesInfo`
- **定时器**：`PData.timerLinesInfo`
- **堆垛**：`PData.stackLinesInfo`
- **点位**：`PData.pointLinesInfo`

**收集函数**：`collectSpecialLines(programIndex)`

```javascript
function collectSpecialLines(programIndex){
    var model = PData.programs[programIndex];
    
    // 清除旧的行信息
    PData.counterLinesInfo.clear(programIndex);
    PData.timerLinesInfo.clear(programIndex);
    PData.stackLinesInfo.clear(programIndex);
    PData.pointLinesInfo.clear(programIndex);
    
    // 遍历所有动作，收集特殊行信息
    for(var i = 0; i < model.count; ++i){
        var actionObject = model.get(i).mI_ActionObject;
        
        // 检查并记录计数器
        if(Teach.hasCounterIDAction(actionObject)){
            var counterIDs = Teach.actionCounterIDs(actionObject);
            for(var c in counterIDs){
                PData.counterLinesInfo.add(programIndex, counterIDs[c], i);
            }
        }
        
        // 检查并记录定时器
        if(Teach.hasTimerIDAction(actionObject)){
            var timerIDs = Teach.actionTimerIDs(actionObject);
            for(var t in timerIDs){
                PData.timerLinesInfo.add(programIndex, timerIDs[t], i);
            }
        }
        
        // 检查并记录堆垛
        if(Teach.hasStackIDAction(actionObject)){
            var stackID = Teach.actionStackID(actionObject);
            PData.stackLinesInfo.add(programIndex, stackID, i);
        }
        
        // 检查并记录点位
        if(Teach.canActionUsePoint(actionObject)){
            var points = Teach.currentRecord.definedPoints
                .parseActionPointsHelper(actionObject);
            for(var p = 0; p < points.length; ++p){
                PData.pointLinesInfo.add(programIndex, points[p].index, i);
            }
        }
    }
}
```

### 3.2 行号同步

当插入或删除指令时，需要同步更新特殊行信息：

```javascript
// 插入后同步（从插入位置开始，所有行号+1）
PData.counterLinesInfo.syncLines(programIndex, insertLine, 1);
PData.timerLinesInfo.syncLines(programIndex, insertLine, 1);
PData.stackLinesInfo.syncLines(programIndex, insertLine, 1);
PData.pointLinesInfo.syncLines(programIndex, insertLine, 1);

// 删除后同步（从删除位置开始，所有行号-1）
PData.counterLinesInfo.syncLines(programIndex, deleteLine, -1);
// ... 其他同理
```

## 七、总结

### 未保存状态
- **存储位置**：内存中的 QML ListModel (`PData.programs`)
- **数据状态**：临时的、易失的
- **标记**：`hasModify = true`

### 已保存状态
- **存储位置**：通过 C++ 控制器持久化到模具数据库/配置文件
- **数据状态**：永久的、持久的
- **标记**：`hasModify = false`
- **格式**：JSON 字符串

### 关键点
1. 插入操作只修改内存中的数据模型
2. 保存操作将内存数据转换为JSON并持久化
3. 系统通过 `hasModify` 标志跟踪保存状态
4. 特殊指令（计数器、定时器等）的行号信息会被单独跟踪
5. 所有操作都会记录到操作日志中

## 菜单状态模式

主菜单根据当前编辑模式有不同的状态：

### 1. 默认模式（主程序编辑）
- 所有菜单项可用
- 条件菜单默认禁用

### 2. 模块编辑模式 (moduleEditMode)
- 条件菜单启用
- 其他菜单项正常

- 堆垛菜单禁用
- 计数器菜单禁用

### 4. 子程序编辑模式 (subProgramEditMode)
- 轴动作菜单禁用
- 路径菜单禁用
- 其他菜单项正常

## 动作编辑器加载机制

### 懒加载策略

动作编辑器采用懒加载策略，只有在首次点击菜单项时才创建对应的编辑器：

```javascript
actionMenuObject.axisMenuTriggered.connect(function(){
    if(axisMenuIndex < 0){  // 首次加载
        var editor = Qt.createComponent('AxisActionEditor.qml');
        var axisEditorObject = Utils.icCreateObject(editor, actionEditorContainer);
        axisMenuIndex = actionEditorContainer.size();
        actionEditorContainer.addPage(axisEditorObject);
    }
    actionEditorContainer.setCurrentIndex(axisMenuIndex);  // 切换到编辑器
    
    // 设置快捷链接
    linkedBtn1.text = qsTr("Output Action");
    linkedBtn1.visible = true;
    PData.linked1Function = actionMenuObject.outputMenuTriggered;
    // ...
});
```

### 优点

1. **节省内存**: 只加载用户实际使用的编辑器
2. **加快启动**: 程序启动时不需要加载所有编辑器
3. **按需创建**: 用户点击时才创建，响应更快

## 插入流程

### 完整流程

```
用户点击主菜单项（如"轴动作"）
  ↓
触发信号 axisMenuTriggered()
  ↓
ProgramActionMenuFrame 接收信号
  ↓
懒加载 AxisActionEditor.qml（如果未加载）
  ↓
切换到轴动作编辑器
  ↓
显示快捷链接按钮
  ↓
用户配置动作参数
  ↓
用户点击"插入"按钮
  ↓
触发 insertActionTriggered() 信号
  ↓
ProgramFlowPage.onInsertTriggered()
  ↓
调用 actionEditorFrame.item.createActionObjects()
  ↓
调用当前编辑器的 createActionObjects()
  ↓
返回 [actionObject]（通常1条指令）
  ↓
_InsertPrograms([actionObject])
  ↓
insertActionToList(actionObject)
  ↓
插入到 ListModel
```

## 与工艺模块的关系

### 工艺菜单项的特殊性

**按钮**: 工艺 (Technology)

**动态文本**: 根据当前工艺类型显示不同的文本
```javascript
onVisibleChanged: {
    if(visible) {
        techno.text = ProductData.getTechTypeName(panelRobotController.getTechnologyType())
    }
}
```

**可能的文本**:
- "None" - 无工艺
- "weldingProcess" - 焊接工艺
- "SpryingProcess" - 喷涂工艺
- "StampingProcess" - 冲压工艺
- "ThineSectionProcess" - 取片工艺（薄板工艺）
- "EasyStackingProcess" - 简易堆垛工艺
- "InjectigProcess" - 注塑工艺
- "DiecastingProcess" - 压铸工艺

**加载的编辑器**: `ProcessMenu.qml`

`ProcessMenu.qml` 是一个工艺路由器，它根据当前工艺类型动态加载对应的工艺模块：
- 取片工艺 → `extents/ThinSectionCraft.qml`
- 喷涂工艺 → `SpecialSprayTechnology.qml`
- 冲压工艺 → `StampingTechnology.qml`
- 其他 → `EmptyTechnologySettingPage.qml`

### 工艺模块与通用动作的区别

| 特性 | 通用动作 | 工艺模块 |
|-----|---------|---------|
| **菜单入口** | 独立菜单项 | 统一的"工艺"菜单项 |
| **编辑器** | 固定的编辑器文件 | 根据工艺类型动态加载 |
| **指令数量** | 通常1条 | 通常几十条（完整流程） |
| **参数配置** | 单个动作的参数 | 整套工艺流程的参数 |
| **生成方式** | 直接创建动作对象 | 根据参数生成指令序列 |

## 动作菜单总结

通用指令插入的动作菜单具有以下特点：

1. **丰富的动作类型**: 提供 20 种不同类型的动作，覆盖机器人编程的各个方面
2. **懒加载机制**: 按需加载编辑器，节省资源
3. **快捷链接**: 提供相关动作的快速切换，提高编程效率
4. **状态模式**: 根据编辑模式动态启用/禁用菜单项
5. **工艺集成**: 通过"工艺"菜单项统一入口，支持多种工艺类型
6. **模块化设计**: 每个动作类型有独立的编辑器，便于维护和扩展

与取片工艺插入流程相比，通用指令插入更加灵活，适合逐条配置指令，而工艺模块则更加自动化，适合批量生成完整的工艺流程。
